# 物流夜班點貨 PWA v7

Android Chrome 自用、離線優先的物流載具點貨工具。資料保存在手機瀏覽器的 IndexedDB，不需要登入或雲端資料庫。

## v7 主要更新

1. 快速分類統一為：
   - NS
   - TS
   - CS
   - S（SS＋KS）
   - E（ET＋HT＋YT）
2. 現場盤點、統計、中班、過境、提前載走的快速分類選單與顯示皆採相同規則。
3. 中班數量改用站所預設載具：
   - 正常操作直接調整預設載具。
   - 「下一筆改用非預設載具」完成一次後自動關閉。
   - 按＋或−會跳出 1、2、5 與自訂數量。
   - 點中央數字可直接輸入最後數量。
4. 提前載走改用站所預設載具，並保留下一筆非預設載具。
5. 新增 05:00 快速回報，只顯示 NS、TS、E（ET／HT／YT）。
6. 第十滑道順序改為 NS23 後顯示 YT1、HT1；第十一滑道在 TS11 後顯示 ET3。
7. 統計頁上方個別系列小框順序：NS、TS、CS、SS、KS、HT、YT、ET。

## 安裝／更新

### 從 v6 更新

1. 先在 v6 的「班次／備份」匯出完整 JSON 備份。
2. 將 v7 所有檔案覆蓋原本網站空間的舊檔案。
3. 完全關閉已安裝的 PWA 與 Chrome 分頁。
4. 重新開啟 App；左上選單底部應顯示「版本 v7.0」。
5. 若仍看到舊版，關閉後再開一次；Service Worker 會更新離線快取。

既有班次、事件、回倉紀錄與載具分類會保留。舊版若停留在 SS、KS、ET、HT、YT 的單獨篩選，升級後會回到「全部站所」，不影響原始資料。

## 電腦本機測試

在包含 `index.html` 的資料夾中執行：

```bat
py -m http.server 8080
```

再用 Chrome 開啟：

```text
http://localhost:8080
```

## 檔案

- `index.html`：App 外殼與對話框
- `styles.css`：Android 直式深色介面
- `logic.js`：站所、載具、統計、回報與匯出邏輯
- `app.js`：操作介面與 IndexedDB
- `manifest.json`：PWA 安裝設定
- `service-worker.js`：離線快取
- `icons/`：App 圖示
- `tests/`：核心與專案靜態測試

## 使用安全

僅在電拖完全停止後操作手機。
