# 物流夜班點貨 PWA v5

Android Chrome 可安裝、可離線使用的個人點貨工具。v5 正式將數量分成「籠車／棧板」，但夜班高頻操作仍維持按站所一次。

## v5 主要更新

1. 底部主要功能改為：夜班、線上、盤點、統計；提前載走移入左上「其他功能」。
2. 早班盤點依字母系列排列：CS、ET、HT、KS、NS、SS、TS、YT。
3. 回倉不必輸入時間；每次＋1會依手機時間自動歸入30分鐘區間，例如00:00–00:30。−1會撤銷該來源、該載具最近一筆，並修正原時段。
4. 線上未退出：0顯示綠框與「已完成」；大於0顯示橘框與「轉完成」。
5. 第十滑道新增YT1、HT1；第十一滑道新增ET3。
6. 點貨、過境、載走、線上、盤點、統計、回報、事件及匯出均支援籠車／棧板細分。

## 預設籠車站所

NS2、NS6、NS8、NS9、NS18；CS12、CS4、CS5、CS6；TS1、TS2、TS5、TS6、TS11；全部SS、KS；YT1、HT1、ET3。

其餘站所預設棧板。

## 夜班操作

- 正常：直接按站所，依該站預設載具＋1。
- 偶發另一種載具：先按「下一筆改用非預設載具」，再按站所；完成一筆後自動恢復預設。
- 修正較早紀錄：進入「修正前面數量」，可分別調整籠車與棧板。

## 線上操作

- 「全部28站＋1」依各站預設載具增加。
- 個別例外可使用「下一筆改用非預設載具」。
- 轉完成會把該站籠車與棧板的線上數量全部移到夜班完成。

## 從v4更新

1. 先在舊版匯出完整JSON備份。
2. 將v5所有檔案覆蓋原網站檔案。
3. 完全關閉已安裝App，再重新開啟。
4. 左上選單底部看到「版本 v5.0」即更新成功。

v4舊事件沒有載具欄位，升級時會依站所預設載具分類。v4舊回倉總數無法判斷載具，會暫列為棧板；需要時可用回倉頁調整，或建立新班次開始正式分類。

## 電腦測試

在包含index.html的資料夾開啟CMD：

```bat
python -m http.server 8080
```

Chrome開啟：

```text
http://localhost:8080
```

## Android安裝

將專案部署到HTTPS靜態網站後，用Android Chrome開啟，選擇「安裝」或「加到主畫面」。首次完整開啟後，可離線使用。

## 資料

資料存於手機Chrome的IndexedDB。清除Chrome網站資料會刪除紀錄，請定期匯出JSON備份。
