# 物流夜班點貨 PWA v11

Android Chrome 可安裝、可離線使用的自用點貨工具。資料只保存在手機本機 IndexedDB，不需登入或雲端資料庫。

## v11 更新

1. 快速回報按下「複製回報文字」時，改為直接檢查「統計核對」的數字差異：
   - 03:00：只檢查 CS 與 S 系列；S 包含 SS、KS。
   - 05:00：只檢查 NS、TS 與 E 系列；E 包含 ET、HT、YT。
2. 籠車或棧板任一載具的統計差異不為 0，就會跳出警告。
3. 警告只顯示「存在數字差異異常」，不再列出站所名稱或詳細差異。
4. 有異常時仍可按「仍要複製回報文字」。
5. 切換 03:00／05:00，或離開快速回報功能後，警告會自動消失。

## 異常檢查與回報文字的差別

- 複製出的 03:00／05:00 回報文字仍只計：中班＋夜班＋過境。
- 複製前的警告則讀取統計核對結果，確認對應範圍內籠車與棧板是否存在數字差異。
- 因此尚未完成盤點、現場數量不符，或籠／板分類不符，都可能觸發警告。

## 從 v10 更新

1. 先在 v10 的「班次／備份」匯出完整 JSON 備份。
2. 將 v11 專案全部檔案覆蓋原網站檔案。
3. 完全關閉已安裝的 App，再重新開啟。
4. 左側選單底部顯示「版本 v11.0」即更新成功。
5. 若仍看到舊版，關閉 App 後再開一次；必要時在 Chrome 重新整理網站，但不要清除網站資料。

## 電腦測試

在含 `index.html` 的資料夾執行：

```bat
py -m http.server 8080
```

Chrome 開啟：

```text
http://localhost:8080
```

## Android 安裝

PWA 必須放在 HTTPS 網址。用 Android Chrome 開啟後，選擇「安裝應用程式」或「加到主畫面」。

## 主要資料公式

- 統計核對回報總數＝中班＋夜班＋過境＋已轉完成二次分理
- 盤點應在現場＝統計核對回報總數－提前載走
- 03:00／05:00快速回報總數＝中班＋夜班＋過境
- 二次分理不列入03:00／05:00快速回報文字
