# 物流夜班點貨 PWA v9

Android Chrome 可安裝、可離線使用的自用點貨工具。資料只保存在手機本機 IndexedDB，不需登入或雲端資料庫。

## v9 更新

1. 回倉來源 `CS12` 改為 `CS4`；舊版 CS12 回倉紀錄會在升級時自動改列 CS4。
2. 中班數量頁新增「複製中班各站所籠／板數」。
3. 回倉點貨面板改為只顯示目前30分鐘時段：
   - 00:00–00:30、00:30–01:00……自動分段。
   - 時段切換後面板自動從0開始。
   - 下方各時段歷史與本班累計仍完整保留。
   - `−1`只扣目前時段同來源、同載具的紀錄，不會誤扣前一時段。
4. 現場盤點輸入框：
   - 點入0時自動清空，方便直接輸入。
   - 未輸入就點到其他地方時恢復為0。
5. 現場盤點預設只顯示站所常用載具；按「＋ 非預設」才展開另一種載具。
   - 若非預設載具的應有數或已輸入數大於0，會自動展開，避免漏點。
6. 盤點卡片右上改為顯示應有 `籠X／板X`。
   - 籠車與棧板兩項都吻合才是綠框。
   - 任一載具不符即顯示紅框，不再顯示差多少。

## 從 v8 更新

1. 先在 v8 的「班次／備份」匯出完整 JSON 備份。
2. 將 v9 專案全部檔案覆蓋原網站檔案。
3. 完全關閉已安裝的 App，再重新開啟。
4. 左側選單底部顯示「版本 v9.0」即更新成功。
5. 若仍看到舊版，關閉 App 後再開一次；必要時在 Chrome 重新整理網站，但不要清除網站資料。

## 電腦測試

在含 `index.html` 的資料夾執行：

```bat
py -m http.server 8080
```

Chrome 開啟：

```text
http://localhost:8080
```

## Android 安裝

PWA 必須放在 HTTPS 網址。用 Android Chrome 開啟後，選擇「安裝應用程式」或「加到主畫面」。

## 主要資料公式

- 回報總數＝中班＋夜班＋過境＋已轉完成二次分理
- 應在現場＝回報總數－提前載走
- 二次分理待處理數在按下「轉完成」前不列入回報或盤點應有數
