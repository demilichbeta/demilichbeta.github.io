# 物流夜班點貨 PWA v4

Android Chrome 使用的離線自用點貨工具。資料只保存在手機本機，不需要登入或雲端資料庫。

## v4 主要調整

1. **復原提示只在夜班頁出現**
   - 切換到其他功能會自動消失。
   - 新增關閉按鈕。
   - 復原只針對剛才在夜班頁完成的操作，不會誤復原其他功能。

2. **新增「修正前面板數」模式**
   - 在夜班頁上方切換。
   - 每個站所提供大型 `−1／＋1`。
   - 可直接修正較早的站所數量，不必逐步復原後面的紀錄。
   - 離開夜班頁或重新開啟 App 時，會回到安全的快速＋1模式。

3. **回倉頁簡化**
   - 只記各來源回倉板數，不記時間、待分／過境分類或備註。
   - 來源：DC9、DC4、DC11、CS12、SDC、DC2、NS2、NS1、DC12。
   - 每個來源都有大型 `−1／＋1` 按鈕。
   - v3舊回倉批次會自動換算成來源總板數。

4. **過境移到其他功能**
   - 底部固定功能改為：夜班、載走、盤點、統計。
   - 過境仍保留完整 `−1／＋1` 功能，統計公式不受影響。

5. **快速回報只保留03:00**
   - 僅顯示CS、SS、KS。
   - 05:00改直接查看統計總表。

6. **滑道名稱更新**
   - 第一～第六滑道改為第七～第十二滑道。

## 滑道與站所

- 第七滑道：NS2、NS3、NS5、NS6、NS8、NS9
- 第八滑道：CS2、CS12、NS10、NS11、NS12、NS13
- 第九滑道：CS4、CS5、NS15、NS16、NS17、NS18、NS19
- 第十滑道：CS3、CS6、NS20、NS21、NS22、NS23
- 第十一滑道：TS1、TS2、TS3、TS5、TS6、TS11
- 第十二滑道：SS3、SS4、SS5、SS6、SS7、KS1、KS2、KS3

## 統計公式

- 回報總數＝早班＋夜班＋過境＋線上
- 應在現場＝回報總數－提前載走
- 差異＝現場實際－應在現場

## 從v3更新到v4

1. 先在v3的「班次／備份」匯出完整JSON備份。
2. 將v4全部檔案覆蓋原網站空間。
3. 完全關閉手機上的舊App，再重新開啟。
4. 左上選單底部應顯示「版本 v4.0」。
5. 不要清除Chrome網站資料，否則本機紀錄會被刪除。
6. 若仍顯示舊版，關閉App再開一次，或在Chrome重新載入網站。

v3舊回倉資料會依來源自動合併，數量以原批次的「待分＋過境」作為回倉總板數。

## 電腦本機測試

1. 解壓縮ZIP。
2. 進入包含 `index.html` 的資料夾。
3. 雙擊 `START_LOCAL_SERVER.bat`，或在CMD執行：

```bat
py -m http.server 8080
```

4. Chrome開啟：

```text
http://localhost:8080
```

## Android安裝

PWA必須放在HTTPS網站空間後，才能在Android Chrome完整安裝與離線使用。用Chrome開啟網站後，按畫面右上角「安裝」或Chrome選單中的「加到主畫面」。

## 資料安全

- 每次操作自動寫入IndexedDB。
- 更新前先匯出JSON備份。
- 清除Chrome網站資料會連同App資料一起刪除。
- App不會把工作資料傳到雲端。
