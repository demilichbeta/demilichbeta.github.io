# 物流夜班點貨 PWA v10

Android Chrome 可安裝、可離線使用的自用點貨工具。資料只保存在手機本機 IndexedDB，不需登入或雲端資料庫。

## v10 更新

1. 中班數量複製文字改為「數字在前」：
   - `NS2：2籠`
   - `NS3：3板`
   - `TS1：1籠／1板`
   - 合計格式：`3籠／4板｜總7`
2. 03:00與05:00快速回報移除「二分」：
   - 畫面不再顯示二分欄。
   - 複製文字不再出現二分。
   - 快速報告總數只計算中班＋夜班＋過境。
3. 複製快速回報前增加異常檢查：
   - 沒有異常時直接複製。
   - 有異常時先顯示紅色提示框，列出異常站所與籠／板核對狀況。
   - 異常時不會立即複製，可選擇「仍要複製回報文字」。
   - 切換03:00／05:00或移到其他功能後，提示框自動消失。
4. 盤點輸入0也能被視為已完成輸入：若應有數不為0，快速回報檢查可以正確提示異常。

## 快速回報異常判定

- 尚未做任何現場盤點的站所，不會被全部誤判為異常。
- 只要該站已輸入過盤點數量，就會分別核對籠車與棧板。
- 籠車或棧板任一項不符，提示框會列出該站，例如：`籠應2／現1`。
- 非整數、負數或損壞資料也會被標示為數值異常。

## 從 v9 更新

1. 先在 v9 的「班次／備份」匯出完整 JSON 備份。
2. 將 v10 專案全部檔案覆蓋原網站檔案。
3. 完全關閉已安裝的 App，再重新開啟。
4. 左側選單底部顯示「版本 v10.0」即更新成功。
5. 若仍看到舊版，關閉 App 後再開一次；必要時在 Chrome 重新整理網站，但不要清除網站資料。

## 電腦測試

在含 `index.html` 的資料夾執行：

```bat
py -m http.server 8080
```

Chrome 開啟：

```text
http://localhost:8080
```

## Android 安裝

PWA 必須放在 HTTPS 網址。用 Android Chrome 開啟後，選擇「安裝應用程式」或「加到主畫面」。

## 主要資料公式

- 統計核對回報總數＝中班＋夜班＋過境＋已轉完成二次分理
- 盤點應在現場＝統計核對回報總數－提前載走
- 03:00／05:00快速回報總數＝中班＋夜班＋過境
- 二次分理不列入03:00／05:00快速回報
