# 夜班點貨 PWA（NS／TS v2）

Android 自用、離線優先的棧板點貨 App。資料保存在手機 Chrome 的 IndexedDB，不需登入、不使用雲端資料庫，也不連接公司系統。

## v2 本次修改

1. **過境貨改為大型 −1／+1**
   - 不再顯示 +2、+3、+5 或自訂數量。
   - 過境數量不允許低於 0。
   - 誤按仍可使用「復原上一筆」。

2. **線上未退出增加整批操作**
   - 「全部 25 站＋1」：一次替所有 NS／TS 站所的線上數量加一。
   - 「全部線上轉完成」：把目前所有線上板一次轉入夜班完成數。
   - 兩種整批操作都屬於同一個操作，可用「復原上一筆」整批撤回。

3. **早班盤點改為手機大型控制**
   - 每站提供大型「−／＋」按鈕。
   - 中間數字仍可點擊，使用 Android 數字鍵盤直接輸入。
   - 數量不允許低於 0。

## 既有功能

- 25 個站所依五個滑道順序排列。
- 夜班站所按一下直接 +1，包含震動、成功效果及時間紀錄。
- 過境、提前載走、線上、早班及現場盤點。
- 線上單站 −1、+1、轉完成。
- 現場盤點與即時差異。
- NS、TS、全部回報總數。
- 只看 NS、只看 TS、只看異常站所。
- 事件篩選、排序、修改及刪除。
- 每日獨立班次及歷史班次切換。
- CSV、完整 JSON 備份、JSON 還原、工作日誌文字複製。
- Service Worker 離線快取及 PWA 安裝。
- 可選擇保持螢幕喚醒。

## 統計公式

- 回報總數＝早班＋夜班＋過境＋線上
- 應在現場＝回報總數－提前載走
- 差異＝現場實際－應在現場

## 專案檔案

- `index.html`：App 主頁
- `styles.css`：Android 直式深色介面
- `logic.js`：站所、事件及統計邏輯
- `app.js`：畫面、IndexedDB、匯出與互動

- `service-worker.js`：離線快取

- `TEST_RESULTS.txt`：本版自動檢查結果

## 電腦測試

不要直接雙擊 `index.html`。在包含 `index.html` 的資料夾開啟 CMD：

```bat
python -m http.server 8080
```

若 `python` 指令無法使用：

```bat
py -m http.server 8080
```

再用 Chrome 開啟：

```text
http://localhost:8080
```

## Android 手機測試

PWA 必須從 HTTPS 網址開啟，才能正常安裝與離線使用。最簡單的免費方式是 GitHub Pages：

1. 建立 GitHub 儲存庫。
2. 將 ZIP 內 `pallet-counter-pwa` 資料夾中的全部檔案，上傳至儲存庫根目錄。
3. 前往儲存庫 `Settings → Pages`。
4. 選擇從主要分支的根目錄發布。
5. 等待 GitHub 產生 HTTPS 網址。
6. Android Chrome 開啟該網址。
7. 按 App 右上角「安裝」，或 Chrome 選單的「加到主畫面」。
8. 第一次完整開啟後，再測試飛航模式離線啟動。

程式檔公開不代表工作資料公開；本版沒有把點貨資料上傳，紀錄只存在該手機的瀏覽器資料庫。

## 從 v1 更新到 v2

- 本版保留相同資料結構，原本 v1 的班次與事件資料可以繼續使用。
- 部署新檔案後，完全關閉 App 再重新開啟。
- Service Worker 快取名稱已更新為 `night-pallet-counter-v2.0.0`。
- 若畫面仍是舊版，先等待約十秒後關閉重開；不要先清除網站資料，以免刪除尚未備份的點貨紀錄。
- 更新前建議先匯出完整 JSON 備份。

## 資料安全

- 每天下班建議匯出一次完整 JSON。
- CSV 是目前班次事件清單；JSON 才能完整還原全部班次。
- 清除 Chrome 網站資料或解除安裝並清除資料，可能刪除所有本機紀錄。
- 換手機前務必先匯出 JSON，再於新手機匯入。

## 自動測試

```bash
node tests/test-project.js
```

## 現場安全

所有操作均以電拖完全停止為前提，不要在行駛中操作手機。
