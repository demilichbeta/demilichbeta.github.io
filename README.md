# 物流夜班點貨 PWA v3

這是一款供 Android Chrome 使用的離線自用點貨工具。資料只保存在手機瀏覽器的 IndexedDB，不需要登入，也不會上傳雲端。

## v3 新增內容

1. 加入 CS、SS、KS 站所，依現場滑道順序排列。
2. 新增「03:00／05:00 快速回報」頁：
   - 03:00：CS、SS、KS。
   - 05:00：NS、TS。
   - 顯示早班、夜班、過境及總數。
   - 可一鍵複製回報文字。
3. 新增「回倉／過境批次」頁：
   - 自動記錄新增時間。
   - 記錄來源、待分板數、過境板數及備註。
   - 快速選擇 DC9、DC4。
   - 可修改或刪除紀錄。
   - 比較「回倉批次過境」與「各站所過境加總」的差異。
4. 班次頁新增「匯出回倉 CSV」。
5. 支援從 v2 原地升級：使用相同本機資料庫，舊 NS／TS 事件及班次會保留，新站所從 0 開始。
6. Service Worker 升級為 v3 快取，更新網站檔案後較容易取得新版。

## 重要計算公式

- 回報總數＝早班＋夜班＋過境＋線上。
- 應在現場＝回報總數－提前載走。
- 差異＝現場實際－應在現場。
- 回倉過境差異＝各站所過境加總－回倉批次過境總數。

## 從 v2 更新到 v3

更新前建議先在「班次／備份」匯出完整 JSON。

1. 將 v3 專案全部檔案覆蓋原網站檔案。
2. 在 Android 上完全關閉已安裝的點貨 App。
3. 重新開啟 App，等待約數秒。
4. 若仍顯示舊版，再完全關閉並開啟一次。
5. 打開左上角選單，底部應顯示「版本 v3.0」。
6. 舊資料應保留；新 CS／SS／KS 數量初始為 0。

不要先清除 Chrome 網站資料，否則本機資料會被刪除。

## 電腦測試

在包含 `index.html` 的資料夾中開啟 CMD，執行：

```bat
python -m http.server 8080
```

若找不到 `python`，使用：

```bat
py -m http.server 8080
```

Chrome 開啟：

```text
http://localhost:8080
```

停止伺服器：在 CMD 按 `Ctrl + C`。

## Android 安裝

PWA 必須放在 HTTPS 網址上，Android Chrome 才能完整安裝及離線使用。

1. 將專案部署到原本使用的 HTTPS 網站空間。
2. Android Chrome 開啟網址。
3. 點網頁內「安裝」，或 Chrome 選單的「安裝應用程式／加到主畫面」。
4. 首次開啟時保持網路，讓所有檔案完成快取。
5. 關閉網路後重新開啟，確認可離線使用。

## 回倉批次的使用方式

例如 01:05 DC9 回來 10 板待分、2 板過境：

1. 選單 →「回倉／過境批次」。
2. 點 DC9。
3. 待分輸入 10，過境輸入 2。
4. 點「現在時間新增回倉紀錄」。
5. 再到「過境」頁，把那 2 板分別加到實際站所。
6. 回倉頁的「過境差異」回到 0，代表批次總數與站所明細一致。

## 資料安全

- 每次點擊與輸入都會自動保存。
- 資料位於瀏覽器本機。
- 清除網站資料、解除安裝後清除資料，或手機損壞，可能造成資料消失。
- 建議每班或每週匯出一次完整 JSON。
- 匯入 JSON 會取代目前全部資料，操作前會要求確認。

## 專案檔案

- `index.html`：頁面結構。
- `styles.css`：手機介面。
- `logic.js`：站所、計數、公式、回報、回倉與匯出邏輯。
- `app.js`：畫面、IndexedDB、操作事件與PWA安裝。
- `manifest.json`：PWA安裝設定。
- `service-worker.js`：離線快取。
- `icons/`：App圖示。
- `tests/`：Node.js自動測試。
- `TEST_CHECKLIST.md`：Android實機測試項目。
- `TEST_RESULTS.txt`：本次自動測試結果。

## 安全提醒

只能在電拖完全停止後操作手機。
